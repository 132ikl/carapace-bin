delete
Delete resources by filenames, stdin, resources and names, or by resources and label selector
--all   Delete all resources, including uninitialized ones, in the namespace of the specified resource types
-A, --all-namespaces   If present, list the requested object(s) across all namespaces. Namespace in current context is igno
--cascade VAL  Must be "background", "orphan", or "foreground". Selects the deletion cascading strategy for the dep
--dry-run VAL  Must be "none", "server", or "client". If client strategy, only print the object that would be sent,
--field-selector VAL  Selector (field query) to filter on, supports '=', '==', and '!='.(e.g. --field-selector key1=value1
-f, --filename VAL  containing the resource to delete.
--force   If true, immediately remove resources from API and bypass graceful deletion. Note that immediate del
--grace-period VAL  Period of time in seconds given to the resource to terminate gracefully. Ignored if negative. Set to
--ignore-not-found   Treat "resource not found" as a successful delete. Defaults to "true" when --all is specified.
-k, --kustomize VAL  Process a kustomization directory. This flag can't be used together with -f or -R.
--now   If true, resources are signaled for immediate shutdown (same as --grace-period=1).
-o, --output VAL  Output mode. Use "-o name" for shorter output (resource/name).
--raw VAL  Raw URI to DELETE to the server.  Uses the transport specified by the kubeconfig file.
-R, --recursive   Process the directory used in -f, --filename recursively. Useful when you want to manage related man
-l, --selector VAL  Selector (label query) to filter on, not including uninitialized ones.
--timeout VAL  The length of time to wait before giving up on a delete, zero means determine a timeout from the siz
--wait   If true, wait for resources to be gone before returning. This waits for finalizers.
